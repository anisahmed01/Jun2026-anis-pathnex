# Day 22 
**Overall Agenda:** Use Ansible handlers to restart nginx only when its config actually changes. Create a security group controlling inbound traffic. Run a monitoring process on every node in the cluster using a DaemonSet. Watch disk usage in a shell script. Trigger builds automatically only when a Git tag is pushed.

## Section 1: Ansible | YAML | ``.yml``
**Goal:**  copy a custom nginx config to all servers and restart nginx only if the file actually changed
```
- name: Configure Nginx for Blinkit
  hosts: all
  become: yes
  tasks:
    - name: Copy Nginx config
      copy:
        dest: /etc/nginx/nginx.conf
        content: |
          user nginx;
          worker_processes auto;
      notify: Restart nginx
  handlers:
    - name: Restart nginx
      service:
        name: nginx
        state: restarted
```

## Section 2 | Terraform | HCL |``.tf``
**Goal:** create a security group inside a VPC that allows HTTP traffic on port 80
```
resource "aws_security_group" "BlinkitSG" {
  name   = "blinkit-sg"
  vpc_id = aws_vpc.BlinkitVPC.id
  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }
  tags = {
    Name = "Blinkit-SG"
  }
}
```

## Section 3 | Kubernetes | YAML ``.yml``
**Goal:** run a monitoring container on every single node in the cluster automatically
```
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: blinkit-daemon
spec:
  selector:
    matchLabels:
      app: ds
  template:
    metadata:
      labels:
        app: ds
    spec:
      containers:
        - name: monitor
          image: busybox
          command: ["sh", "-c", "while true; do echo Blinkit Node Monitor; sleep 20; done"]
```

## Section 4 | Shell Script | Bash | ``.sh``
**Goal:** extract and display the current disk usage percentage of the root partition
```
#!/bin/bash
USAGE=$(df -h / | tail -1 | awk '{print $5}')
echo "Disk Usage: $USAGE"
```

## Section 5 | Jenkins | Groovy | ``Jenkinsfile``
**Goal:** poll the repo every 15 minutes and run the build stage only when a Git tag is pushed
```
pipeline {
    agent any
    environment {
        INSTITUTE_NAME = "Pathnex"
        TEAM           = "DevOps"
        ENV            = "prod"
    }
    triggers {
        pollSCM('H/15 * * * *')
    }
    stages {
        stage('Checkout') {
            steps {
                git url: 'https://github.com/Pathnex/sample-java-app.git', branch: 'main'
            }
        }
        stage('Build') {
            when {
                buildingTag()
            }
            steps {
                sh 'echo "Building $INSTITUTE_NAME app for $TEAM team in $ENV environment"'
            }
        }
    }
}
```

## Section 6 | GitLab CI/CD | YAML | ``.gutlab-ci.yml``
**Goal:** run the build job only when a tag is pushed, not on every commit
```
stages:
  - build

variables:
  INSTITUTE_NAME: "Pathnex"
  TEAM: "DevOps"
  ENV: "prod"

build:
  stage: build
  image: maven:3.8.1-jdk-17
  script:
    - echo "Building $INSTITUTE_NAME app for $TEAM team in $ENV environment"
  only:
    - tags
```

### What happens end to end
- Ansible copies the nginx config file and then waits — the handler only fires if the file actually changed, so nginx doesn't get restarted unnecessarily every time the playbook runs
- The security group acts as a firewall rule attached to the VPC — only port 80 HTTP traffic is allowed in, everything else is blocked by default
- The DaemonSet ensures the monitoring container is always running on every node — when a new node joins the cluster, Kubernetes automatically puts one copy of it there too without any manual work
- The shell script checks how full the disk is in one line — useful as a cron job that alerts when storage is getting low before it becomes a problem
- Jenkins polls the repository every 15 minutes but the build stage only actually runs when a Git tag is detected — normal commits are checked out but skipped, tags trigger the full build
- GitLab does the same with ``only: tags`` — the job is completely ignored on regular commits and only activates when someone pushes a version tag like ``v1.0.0``
